# 赛博朋克首页房间布局规范

> 本文是首页 3D 房间布局的当前基准。后续调整 `src/components/cyberpunk/sceneLayout.ts`、家具组件或替换 GLB 模型时，应优先保持本文定义的空间关系。

## 方向与坐标

房间布局统一使用东南西北和上下，不使用依赖相机视角的方向描述。

| 方向 | Three.js 坐标 | 说明 |
|------|---------------|------|
| 东 | `+X` | 东墙，床、衣柜和书柜所在方向 |
| 西 | `-X` | 西墙，默认相机偏西南 |
| 南 | `+Z` | 房间入口和默认相机所在方向 |
| 北 | `-Z` | 北墙与三块窗所在方向 |
| 上 | `+Y` | 天花板方向 |
| 下 | `-Y` | 地面方向 |

## 布局原则

1. 工作区和服务器必须组成同一个技术区，服务器应靠近工作桌、显示器、键盘和主机线缆，不放到床或衣柜旁边。
2. 床和衣柜必须组成同一个生活区，二者沿东墙邻近布置，形成睡眠、换衣和收纳的一组空间。
3. 书柜是内容墙，不归入服务器区。书柜放在东北或东墙偏北，作为博客合集、阅读和收藏入口。
4. 中部保留客厅核心区，包含地毯、茶几和文章终端。文章终端是内容入口，不应被沙发、床或书柜遮挡。
5. 沙发放在南中偏西，面向中部文章终端。后续在线用户占位优先发生在沙发区域，不进入床区。
6. 从西南入口/默认相机区域到中部，再到东侧生活区，必须保留清晰通道。

## 分区基准

| 区域 | 推荐位置 | 主要元素 | 约束 |
|------|----------|----------|------|
| 工作技术区 | 西北 | 工作桌、三联屏、键盘、椅子、服务器/主机柜、短线缆 | 服务器贴近工作桌，不与床区混放 |
| 内容书柜区 | 东北 | 书柜、合集书本、数据核心、阅读物 | 与服务器区分离，靠近内容动线 |
| 中央客厅区 | 中部略偏西 | 地毯、茶几、文章终端 | 保持可见和可点击，不挤压通道 |
| 沙发等待区 | 南中偏西 | 沙发、在线用户坐席、边几 | 面向文章终端，与床保持明显距离 |
| 睡眠生活区 | 东中至东南 | 床、床头柜、衣柜、衣架 | 床与衣柜邻近，形成私密生活带 |
| 入口/默认相机区 | 西南 | 默认相机、进入动线 | 不放高大家具，避免挡住默认视角 |

## 推荐动线

动线从西南进入，先经过南侧沙发边缘，再到中部文章终端，最后转向东侧床和衣柜。实现时应避免在通道上放置高柜、服务器、床头柜或大型装饰。

## 代码布局基准

`sceneLayout.ts` 应保持以下空间关系：

| 物件 | 调整目标 |
|------|----------|
| `desk` / `monitors` / `chair` | 保持西北工作区 |
| `serverRack` | 保持在西北工作区附近，贴近工作桌 |
| `bookshelf` | 保持在东北或东墙偏北，独立于服务器 |
| `bed` | 保持在东中至东南生活区，沿东墙摆放 |
| `wardrobe` | 靠近床，形成东侧生活区 |
| `coffeeTable` / `articleTerminal` | 中部略偏西，维持客厅核心 |
| `sofa` | 南中偏西，面向文章终端 |

## 校验清单

- 默认相机从西南看向东北时，能同时看到工作区、服务器、书柜、文章终端、沙发和东侧生活区。
- 服务器在视觉和空间上贴近工作区，不靠近床。
- 床和衣柜在同一侧成组出现，和沙发之间有明显缓冲。
- 书柜作为内容墙独立可见，不被服务器或床遮挡。
- 中部文章终端是客厅视觉中心。
- 西南到东侧生活区的通道清晰可读。
