# 项目计划列表

本目录存放项目的各种改进计划和重构方案。

## 📋 当前计划

### [聊天系统简化：从 ReAct 到简单 RAG](./chat-system-simplification.md)

**状态**：📝 计划中

**目标**：移除 ReAct Agent，改用经典 RAG 架构

**核心问题**：
- ReAct 对于单一检索场景是过度设计
- 性能浪费（2 次 LLM 调用 vs 1 次）
- 代码复杂度高（600+ 行 vs 200 行）

**预期收益**：
- 性能提升 50%（响应时间 5s → 2.5s）
- 成本降低 50%（Token 消耗减半）
- 代码量减少 67%（600 行 → 200 行）

**适用场景**：
- 90% 的知识库问答场景
- 单步查询，不需要多步推理
