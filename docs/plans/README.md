## 📋 当前计划

### [RAG 系统重构：简化向量更新机制](./rag-system-refactor.md)

**状态**：📝 计划中

**目标**：简化当前的增量更新逻辑，改为全量更新 + 同步执行

**主要变更**：
- 移除 `TbPostChunk` 和 `TbPostVersion` 表
- 移除增量更新逻辑（Hash 对比）
- 修复文本清理问题（保留代码块）
- 同步执行向量化，失败时明确反馈

**预期收益**：
- 代码减少 70%（1500+ 行 → ~500 行）
- 提高可靠性和搜索质量
- 降低维护成本

**代价**：
- API 响应时间增加约 1 秒
