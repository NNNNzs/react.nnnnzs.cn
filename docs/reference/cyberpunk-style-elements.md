> **用途**: 为 `NNNNzs` 夜间模式、`/chat` 夜间回答风格、首页 3D 房间和文章日志流提供风格词库与设计母题。

## 使用原则

本文档是灵感整理，不是要求直接复制某个游戏、影视或品牌的专有表达。

夜间模式可以吸收赛博朋克的语义结构：记忆被数据化、城市被企业和广告覆盖、身体与机器互相入侵、孤独的人在终端前寻找旧信号。正式 UI 文案应优先发展 `NNNNzs` 自己的词汇体系。

## 通用赛博朋克范式

| 范式 | 含义 | 可转译到博客 |
|------|------|--------------|
| High tech, low life | 极高技术与低处生活并置 | 高级 AI / 3D / 向量检索背后仍是一个人的私人书房 |
| 近未来城市 | 不是遥远太空，而是熟悉城市的技术异化版本 | 博客不是宇宙飞船，是夜里亮着屏幕的小房间 |
| 企业与平台 | 巨型组织掌握基础设施、身份和记忆 | 可转成“系统、协议、接口、云端记忆”的隐喻 |
| 霓虹与雨夜 | 人造光在湿润表面反射，明暗对比强 | 首页夜间窗景、地面反光、文章日志流 |
| 黑色电影感 | 孤独、低声、边缘人物、侦探式追索 | `/chat` 夜间回答可像深夜终端里检索旧档案 |
| 身体/意识改造 | 人的记忆、感知和身体被技术重写 | 回想模块、聊天记录、人生之书的数字分身 |
| 信息即货币 | 数据、身份、访问权限是关键资源 | 搜索、合集、标签、聊天引用都可表现为档案访问 |

## Cyberpunk 2077 可借用母题

### 四种视觉风格

`Cyberpunk 2077` 的 Night City 有四种主要视觉风格，可作为夜间模式的内部参考坐标：

| 风格 | 关键词 | 对本站的启发 |
|------|--------|--------------|
| Entropism | 实用、贫乏、拼装、为生存服务 | 线缆、旧设备、房间杂物、临时修补痕迹 |
| Kitsch | 高饱和、反叛、霓虹、植入体展示 | 文章日志流的青粉边框、夜间按钮、状态芯片 |
| Neomilitarism | 克制、企业、黑色、功能主义、压迫感 | 终端面板、权限页面、后台工具的硬朗边界 |
| Neokitsch | 奢华、明星、金属、木材和大理石 | 不作为主方向，可少量用于“珍贵记忆/收藏”区域 |

本站夜间主方向建议是 **Entropism + Kitsch + 少量 Neomilitarism**。不要整体走 Neokitsch，否则会从私人博客变成奢华夜店。

### 意识、记忆与死亡相关词

| 词 | 资料含义 | 可转译到本站 |
|----|----------|--------------|
| Relic | Arasaka 的生物芯片，用于保存和读取数字化人格/记忆构造 | `/chat` 回想模块可叫“Relic 回响”或“记忆芯片” |
| Engram | 数字化意识、人格构造、被编码保存的人类意识 | 聊天记录可视为“问答 engram”，但正式中文可写“人格回声” |
| Soulkiller | 把人的意识转成数字构造的程序 | 不建议直接做 UI 名称，过重且负面；适合内部理解“意识上传”的阴影面 |
| Mikoshi | 存放数字人格的设施；词源也可联想到日语“神輿/神轿/便携神龛” | “神龛”适合做夜间回想/归档模块的高级隐喻 |
| Secure Your Soul | 以永生为卖点的意识保存计划 | 可转成“保存旧我”“封存回声”等原创文案 |
| Afterlife | 佣兵和中间人聚集的传奇酒吧，名字有“死后/余生”的双关 | 可转成“深夜入口”“旧事酒馆”，但不宜直接照搬 |

## `/chat` 夜间回答风格

夜间回答应该是“赛博朋克一点”，不是“失去准确性一点”。

建议风格：

- 仍然使用中文，仍然第一人称，仍然基于检索结果回答。
- 句式可以更短、更像终端回声，但不输出乱码、伪代码或过度英文。
- 可以使用“终端、日志、回声、芯片、神龛、雨夜、霓虹、扫描、接入”等意象。
- 引用格式保持不变，尤其是文章链接和来源说明不能因风格变化而丢失。
- 未检索到内容时，夜间空状态可以更像“终端无信号”，但要让用户知道确实没有找到。

示例：

```txt
日间：
这一页似乎还是空白，可能我还需要更多的人生去书写它。

夜间：
终端里暂时没有检索到这段回声。也许这块记忆芯片还没有写入，或者它仍在更深的档案层里沉默。
```

```txt
日间：
我在这些文章里看到一个很清楚的线索：你一直把技术当作整理世界的方式。

夜间：
日志里有一条稳定的信号：技术不是冰冷外壳，而是你用来给混乱世界重新布线的工具。
```

## 视觉元素清单

| 元素 | 建议用法 | 禁忌 |
|------|----------|------|
| 霓虹青/粉 | 用于边框、状态点、关键 CTA、少量高光 | 整页到处发光 |
| 扫描线 | 用于终端面板背景，透明度低 | 遮挡正文阅读 |
| 切角边框 | 用于夜间卡片和日志项 | 所有卡片都切角导致疲劳 |
| 雨夜反光 | 首页 3D 地面、窗景、Banner | 普通文章详情页不强加 |
| 终端编号 | `LOG 01`、`SYNC`、`LINK` | 不要把重要中文信息藏进缩写 |
| 数据芯片 | 标签、合集、引用来源的小徽标 | 不要破坏 SEO 和可访问性 |
| 神龛/封存 | 回想、归档、重要历史内容 | 不适合普通操作按钮 |

## 与日间模式的对应关系

夜间模式不是独立新站，而是同一内容的另一层人格。对应关系要稳定：

| 语义 | 日间 | 夜间 |
|------|------|------|
| 内容空间 | 书房 | 终端室 |
| 文章 | 书页 | 日志 |
| 合集 | 书架 | 数据书架 |
| 标签 | 便签 | 信标 |
| 聊天记录 | 回想 | 记忆芯片 |
| 站长人格 | 写作者 | 深夜 Netrunner |
| 加载中 | 正在翻页 | 正在同步 |
| 未找到 | 这一页空白 | 终端无信号 |

## 风格边界

1. 不要直接使用游戏 logo、UI 图标、台词或专有视觉资产。
2. `Relic`、`Mikoshi`、`Afterlife` 等专名可以作为内部设计参考；面向用户时优先发展原创中文词。
3. 不要把中文博客变成满屏英文终端，主要信息仍然中文表达。
4. 不要为了“赛博朋克”牺牲文章可读性、移动端布局和屏幕阅读器语义。
5. 中国大陆用户访问为主，图片、字体、视频、音频等外部资产不能依赖不可控 CDN。
